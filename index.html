通过hexo创建并发布blog

通过hexo创建并发布blog

step by step

本文记录我用mac系统安装hexo的过程和遇到的问题,hexo虽然相对简单,但是很多地方官方文档和其他博客并没有讲清楚.记录在这里,给需要的朋友,少浪费时间在这上面.
hexo版本3.1.1

  1. 包管理器安装nodejs brew install node
  2. 包管理器安装git brew install git
  3. nodejs安装hexo npm install -g hexo-cli
  4. 在Documents目录下创建hexo博客的目录 hexo init blog

  5. 初始化目录

新建完成后,blog文件夹下的内容
  1. 安装hexo git插件 pm install hexo-deployer-git --save

  2. 配置_config.yml,注意yml的格式,:后面一定要有个空格才能接内容

    该文件的各项含义官网上都有,我只列出我修改的内容,尤其注意最好把root注掉

  3. github中新建repository,Repository name为hoticezy.github.io

  4. 新建博客文章 hexo new first,创建第一个博客文件
在_post文件夹下,会新增一个first.md文件,打开后,前面是yml格式的文件头,列出title,date,tags等信息,接下来是---隔开的正文内容,还可以增加摘要
  1. 生成静态页面hexo g,将markdown格式的文章内容和主题等文件,编译成静态网页,放到public目录下
  2. 预览blog内容hexo s,然后通过浏览器查看
  3. 发布静态页面到githubhexo d , 将文章内容和样式等文件,编译成静态网页,放到public目录下.hexo d的作用是将public目录下的所有内容,上传到https://github.com/hoticezy/hoticezy.github.io这个repository中.github会自动识别内容并呈现到https://hoticezy.github.io/这个url上. 10和12可以合并成一步:hexo g --d.

这样就完成了blog的创建和发布过程.

进阶内容

安装使用主题

在blog文件夹下,执行git clone https://github.com/A-limon/pacman.git themes/pacman,下载pacman主题到themes文件夹下
将主题文件夹放到themes目录下,然后修改_config.yml中theme: pacman

评论框

静态博客要使用第三方评论系统，pacman配置了多说评论系统（/themes/pacman/_config.yml），默认关闭，只要将其打开即可:false->true。直接用你的微博/豆瓣/人人/百度/开心网帐号登录多说，即可发表平评论。
